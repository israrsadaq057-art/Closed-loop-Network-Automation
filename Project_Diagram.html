The Four Stages of Closed-Loop Automation

Click any stage to see detailed explanation

01
DETECT
Ping gateway (192.168.1.1) and DNS (8.8.8.8) every 30 seconds. HTTP/HTTPS checks for web services. SNMP polling for switch/router status.
02
ANALYZE
Count consecutive failures against configured thresholds. Default: 3 failures triggers remediation. Priority mapping for critical devices.
03
DECIDE
Determine appropriate remediation action based on failure type. Gateway issues get highest priority. Cooldown period prevents repeated fixes.
04
ACT
Execute automatic fixes: flush DNS cache (ipconfig /flushdns), renew DHCP lease (ipconfig /release & /renew), restart network adapter.

Automatic Remediation Actions (Click to see details)

Flush DNS Cache

Command: ipconfig /flushdns

When DNS cache becomes corrupted, this command clears it. Used when websites resolve incorrectly or ping fails by name but not by IP.

Renew DHCP Lease

Command: ipconfig /release && ipconfig /renew

When IP address expires or conflicts occur, this releases current IP and requests a new one from DHCP server.

Restart Network Adapter

Command: netsh interface set interface Ethernet disable/enable

Resets the entire network stack. Used when other remediations fail. Last resort before manual intervention.

Decision Tree: When Does Automation Trigger?

Q1: Is Gateway Reachable?
Yes → Monitor continues
No → Count failure +1
Q2: Failure Count >= 3?
No → Continue monitoring
Yes → Trigger remediation
Q3: In Cooldown Period?
Yes → Wait 5 minutes
No → Execute fix
